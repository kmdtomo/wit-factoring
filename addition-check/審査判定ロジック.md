# ファクタリング審査判定ロジック

## 📌 判定の軸

| 軸 | 値 |
|---|---|
| **契約回数** | 初回 / 2回目以降 |
| **希望金額** | 30万円以下 / 30万円超 |
| **信頼状況**（2回目以降のみ） | あり / なし |

---

## 1️⃣ 初回契約

### パターン1-A: 初回 × 30万円以下 → **簡易審査**

#### ✅ 実行する項目

| Phase | 項目 | 目的 |
|-------|------|------|
| Phase 3 | エゴサーチ（申込者） | 犯罪履歴チェック |
| Kintone | 家族構成 | 生活安定性の確認 |
| Kintone | 税金滞納状況 | 支払い能力の確認 |
| Phase 1 | 掛け目 | 買取額/請求額の妥当性 |
| Phase 2 | カバー率 | 担保企業の入金実績 |

#### ⏭️ スキップする項目

- Phase 3: 企業実在性確認（買取企業・担保企業）
- Phase 3: 代表者リスク検索
- Phase 1: 担保謄本の詳細分析

#### 📝 判定基準

- エゴサーチでネガティブ情報なし
- 税金滞納が許容範囲内
- カバー率が基準以上

---

### パターン1-B: 初回 × 30万円超 → **フル審査**

#### ✅ 実行する項目

| Phase | 内容 |
|-------|------|
| Phase 1 | 買取・担保情報処理（全ツール実行） |
| Phase 2 | 通帳分析（全ツール実行） |
| Phase 3 | 本人確認・企業実在性・エゴサーチ（全ツール実行） |
| Phase 4 | 審査レポート生成 |

#### 📝 判定基準

- 全項目を厳格に評価
- Phase 4 のAIレポートで総合判定

---

## 2️⃣ 2回目以降

### 🔍 前提: 信頼状況の判定

#### ✅ 信頼あり（前回問題なし）

以下が満たされている：
- 前回の資金使途が申告通りに使われていた
- 入金のずれがなかった

#### 実行する項目
- 今回契約のカバー率や信憑性
- 担保企業の評価、企業実在性

#### ⏭️ スキップする項目
- 申込者のエゴサーチ等の信頼

**結果**:
- → 低リスク

---

### パターン2-A: 2回目以降 × 信頼あり × 30万円以下 → **最簡易審査**

#### ✅ 信頼あり（前回問題なし）

以下が満たされている：
- 前回の資金使途が申告通りに使われていた
- 入金のずれがなかった

#### 実行する項目

| Phase | 項目 | 目的 |
|-------|------|------|
| Phase 1 | 買取請求書確認 | 買取先が問題ないか
| Phase 2 | カバー率確認 | 担保企業の入金実績のみ |

#### ⏭️ スキップする項目

- Phase 3 全般（エゴサーチ、本人確認、企業実在性）

#### 📝 判定基準

- **緩い判定**: カバー率が問題なければOK
- 軽微なチェックのみ

---

### パターン2-B: 2回目以降 × 信頼あり × 30万円超 → **中程度審査**

#### ✅ 実行する項目

| Phase | 項目 | 目的 |
|-------|------|------|
| Phase 1 | 買取企業の妥当性 | 買取先が問題ないか |
| Phase 1 | 担保企業の妥当性 | 担保先が問題ないか |
| Phase 2 | カバー率 | 入金実績の確認 |
| Phase 2 | **資金使途チェック** | 👇 下記の分岐 |

#### 📌 資金使途による分岐

| 資金使途 | 追加チェック |
|---------|-------------|
| **外注関連** | Phase 2: **入金確認必須**（外注先への支払いが通帳で確認できるか） |
| その他 | 通常のチェック |

#### ⏭️ スキップする項目

- Phase 3: 申込者エゴサーチ
- Phase 3: 本人確認書類チェック

#### 📝 判定基準

- 買取企業・担保企業が妥当
- カバー率が基準以上
- （外注関連の場合）入金が確認できる

---

### パターン2-C: 2回目以降 × 信頼なし → **契約不可 or フル審査**

#### ❌ 契約不可の判定基準

以下の場合は**自動的に契約不可**（または担当者裁量）：
- 入金のずれがある
- 資金用途が不確認
- 手入金の申告（エビデンスが取れるならOK）

#### 🔄 フル審査で再評価

契約不可まではいかないが要注意の場合：
- 初回契約と同等の**フル審査**（パターン1-B）を実行
- 判定基準を厳格化
